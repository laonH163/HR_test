# 외부 watchdog 설정 가이드 (5분 소요)

## 이게 뭔가요?

지금 시스템은 "실행 도중 실패하면" 텔레그램으로 알려줍니다. 하지만 **워크플로 자체가
아예 시작되지 않으면**(GitHub Actions 장애, 스케줄 누락, 리포 설정 사고 등) 알림을 보낼
주체가 없어서 **완전한 무음**이 됩니다. '오늘 브리핑이 안 왔네?'를 사람이 눈치채야 하는
구조입니다.

watchdog은 이걸 **밖에서** 감시합니다. 원리는 단순합니다:

1. 파이프라인이 정상 동작하면(브리핑 발송, 또는 주말·공휴일이라 의도적으로 쉼)
   감시 서비스에 "나 오늘 살아있어요" 신호(ping)를 하나 보냅니다.
2. 감시 서비스는 **정해진 시간까지 신호가 안 오면** 이메일로 알려줍니다.
3. 즉 시스템이 어떤 이유로든 완전히 침묵하면, 시스템 바깥에서 경보가 울립니다.
   (이런 방식을 dead man's switch라고 부릅니다 — '신호가 끊기면 울리는 장치')

코드는 이미 준비돼 있습니다. **URL 하나만 등록하면 켜지고, 등록 안 하면 아무 일도
안 합니다**(지금처럼 동작).

## 설정 방법 (healthchecks.io 무료 플랜 기준)

### 1단계 — 가입

1. https://healthchecks.io 접속
2. 우측 상단 **Sign Up** 클릭 → 이메일 주소 입력 → 받은 메일의 로그인 링크 클릭
   (비밀번호 없이 이메일 링크로 로그인하는 방식)

### 2단계 — 감시 항목(Check) 만들기

1. 로그인하면 보이는 화면에서 **Add Check** 버튼 클릭
2. 이름을 알아보기 쉽게 변경: 연필 아이콘 클릭 → `HR 채용공고 브리핑` 입력
3. **Schedule(일정)** 부분의 연필 아이콘 클릭:
   - **Period**(신호 주기): `1 day` 선택 — 하루에 한 번 신호가 와야 한다는 뜻
   - **Grace Time**(유예): `6 hours` 선택 — 신호가 평소보다 6시간 늦으면 그때 경보.
     (브리핑이 08시니까, 14시까지 신호가 없으면 이메일이 옵니다.
     2차 재시도 실행까지 기다려줄 여유 시간입니다)
4. 저장

### 3단계 — Ping URL 복사

1. 방금 만든 Check를 클릭해서 열면 **Ping URL**이 보입니다
   (`https://hc-ping.com/영문숫자-긴-코드` 형태)
2. 이 URL을 복사합니다. **이 URL은 비밀번호처럼 취급하세요** — 아는 사람은 가짜 신호를
   보낼 수 있습니다. 채팅·문서에 남기지 말 것.

### 4단계 — GitHub에 등록

1. https://github.com/laonH163/HR_test 접속
2. 상단 **Settings** 탭 클릭
3. 왼쪽 메뉴에서 **Secrets and variables** → **Actions** 클릭
4. 초록색 **New repository secret** 버튼 클릭
5. 입력:
   - **Name**: `HEALTHCHECK_PING_URL` (정확히 이대로, 대문자)
   - **Secret**: 3단계에서 복사한 URL 붙여넣기
6. **Add secret** 클릭

끝입니다. 다음 실행부터 자동으로 켜집니다.

### 5단계 — 잘 되는지 확인 (다음 실행 후)

- GitHub Actions 실행 로그에서 `[OK] watchdog ping 발송 완료`가 보이면 정상
- healthchecks.io 대시보드에서 해당 Check가 초록색 **Up** 상태면 정상
- 일부러 확인하고 싶으면: Actions 탭 → Game Finance Job Daily Scraper →
  **Run workflow** 버튼으로 수동 실행 → 몇 분 뒤 대시보드 확인

## 경보가 오면?

이메일 제목이 `DOWN | HR 채용공고 브리핑` 형태로 옵니다. 뜻: **오늘 파이프라인이
완전히 침묵했다** — 텔레그램 크래시 알림조차 못 보낸 상황이므로, GitHub Actions 탭에서
오늘 실행이 있었는지부터 확인하세요:

| 확인 결과 | 의미 | 대처 |
| --- | --- | --- |
| 오늘 실행 자체가 없음 | 스케줄 미발화·리포 설정 문제 | Actions 탭에서 수동 Run workflow, 반복되면 점검 요청 |
| 실행이 도중에 죽어 있음 | 러너 강제 종료 등 (크래시 알림도 못 보낸 경우) | 로그 마지막 부분을 보고 점검 요청 |
| 실행은 성공했는데 경보가 옴 | 텔레그램 발송 실패 또는 ping만 실패 | 로그에서 `[ERR] 텔레그램`/`[WARN] watchdog ping` 검색 |

## 주말·공휴일은?

경보가 오지 않습니다. 파이프라인이 주말·공휴일에 수집을 쉴 때도 "쉬는 게 정상"이라는
ping을 보내도록 되어 있어서, 감시 서비스 입장에서는 신호가 끊긴 적이 없습니다.
