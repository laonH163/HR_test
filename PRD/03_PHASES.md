# GameFinanceScraper -- Phase 분리 계획

> 이 문서는 기능의 복잡도와 의존성에 따라 개발을 단계별로 나누어 구현하는 상세 청사진입니다.
> 각 단계가 끝날 때마다 "실제 동작하는 신뢰성 있는 가동 상태"를 보장하는 것을 원칙으로 합니다.

---

## Phase 1: 데이터 수집 및 SQLite 기반 저장 (MVP 핵심 인프라)

### 목표
- 원티드 및 사람인 채용 플랫폼 API 또는 전용 수집 모듈을 완성합니다.
- 국내 대형 5대 게임사(넥슨, 크래프톤, 엔씨소프트, 넷마블, 카카오게임즈) 공식 채용 사이트의 웹 스크래퍼 기본 틀을 구축합니다.
- 수집한 공고 데이터를 정적 스키마에 맞게 SQLite DB로 안전하게 로컬 저장 및 조회가 가능한 상태를 달성합니다.

### 기능 및 범위
- [ ] 원티드(Wanted) 채무/세무/회계 관련 공고 수집 스크립트 구현
- [ ] 사람인(Saramin) 키워드 검색 기반 크롤러 구현
- [ ] 5대 게임사 공식 채용 페이지 구조 분석 및 전용 정적/동적 크롤러 구현 (BeautifulSoup / Playwright 등)
- [ ] SQLite 데이터베이스 생성 스크립트 작성 및 `job_postings` 원본 데이터 자동 적재 모듈 완성
- [ ] 중복 공고 삽입 방지 로직 (공고 고유 식별값 PK 기준 처리)

### 데이터
- `job_postings` 테이블
- `scrape_logs` 테이블

### 인증
- 불필요 (로컬 크롤러 및 DB 단순 저장 단계)

### "진짜 제품" 체크리스트
- [ ] 임시 목업 데이터가 아닌, 실제 원티드/사람인 및 대형 게임사에 가동되어 실시간 공고 20건 이상 정상 크롤링 확인
- [ ] DB 저장 후 터미널 SQL 쿼리 조회(`sqlite3`) 시 깨짐 없이 한글 데이터 적재 확인
- [ ] 매뉴얼 명령 한 번으로 크롤링부터 저장까지 한 번에 완료

### Phase 1 시작 프롬프트
```markdown
이 PRD를 읽고 Phase 1을 구현해주세요.
@PRD/01_PRD.md
@PRD/02_DATA_MODEL.md
@PRD/04_PROJECT_SPEC.md

Phase 1 범위:
- 원티드 및 사람인 재무/세무/회계 공고 수집 엔진 개발
- 넥슨, 크래프톤 등 5대 대형 게임사 공식 채용 홈페이지 전용 크롤러 초안 구현
- SQLite3 데이터베이스 생성 스크립트 및 수집 데이터 마스터(job_postings) 적재 로직 완성
- 수집 진행 상황을 추적하는 수집 로그(scrape_logs) 기능 구현

반드시 지켜야 할 것:
- 04_PROJECT_SPEC.md의 "절대 하지 마" 목록 준수
- 목업 데이터로 가짜 완료 보고 금지 (실제 실행을 통한 DB 한글 수집 확인 필수)
- 환경변수 및 접근 토큰은 전용 설정이나 .env를 활용할 수 있게 처리
```

## Phase 2: 로컬 하이브리드 분류기 및 변동사항 분석 엔진 (핵심 로직)

### 전제 조건
- Phase 1의 수집 파이프라인이 안정적으로 로컬에서 100% 정상 가동되는 상태

### 목표
- 별도 유료 API 결제 없이 경량 한글 키워드 임베딩 및 규칙 사전(Dictionary)을 가동하여 연봉 범위, 연차 제한, 재택 유무, 회사 규모 등을 정교하게 추출합니다.
- 전날 데이터와 오늘 데이터를 비교 분석하여 공고의 변화 상태(신규, 수정, 마감)를 완전 식별합니다.
- 가독성이 극대화되고 완벽한 반응형 성능을 내는 1회성 단일 파일 대시보드 HTML 템플릿 엔진을 완성합니다.

### 기능 및 범위
- [ ] 정밀 회계/재무/세무/자금 직무 키워드 및 연차 표기 분석 파서 개발
- [ ] 본문 내 '재택', '하이브리드', '리모트', '출근' 키워드와 주변 문장 맥락 분석을 통한 근무형태 3단 분류기 개발
- [ ] 회사명 기준 외부 공공 데이터(예: 크레딧잡, 잡플래닛 공개정보 등) 혹은 사전 등록된 매출 규모 매핑을 통한 회사 매출/사원수 정보 바인딩 모듈 개발
- [ ] 이전 데이터와 당일 수집 데이터를 고유ID 매핑하여 공고 상태(ACTIVE/CLOSED/MODIFIED)를 가려내고 `scrape_logs` 통계를 갱신하는 Delta Analyzer 작성
- [ ] SQLite 테이블에 보관된 누적 데이터를 기반으로 검색, 직무 필터링, 연차 슬라이더 기능이 완벽히 내장된 `index.html` 단일 정적 웹페이지 자동 생성 모듈 제작

### 추가 데이터
- `job_categories` 테이블 (정밀 분류 테이블 본격 가동)

## Phase 3: 프라이빗 텔레그램 연동 및 Github Actions 배포 (무비용 자동화 완료)

### 전제 조건
- Phase 1 + 2의 수집 및 가공, HTML 리포트 생성 로직이 단 한 줄의 수동 조작 명령어로 한 번에 문제없이 돌아가는 완성 상태

### 목표
- 나만 접근할 수 있는 프라이빗 텔레그램 개인 봇을 개설하고 파이썬으로 텔레그램 메시지 API 통신부를 구축합니다.
- 매일 정해진 아침 시간에 스스로 돌며 모든 분석을 마친 후 스마트폰으로 화려하게 정돈된 소식을 알려주는 GitHub Actions 일일 파이프라인을 구축합니다.

### 기능 및 범위
- [ ] `python-telegram-bot` 또는 원시 HTTP request를 활용한 메시지 전송 로직 개발
- [ ] 텔레그램용 아침 요약 브리핑 카드 마크다운 템플릿 구현 (신규 공고, 특이사항이 감지된 수정 공고, 오늘 마감된 공고 링크 한눈에 표시)
- [ ] GitHub Actions 워크플로우 파일(`.github/workflows/daily-scraper.yml`) 설정
  - 크론 스케줄 설정 (매일 한국 시간 오전 8시)
  - 파이썬 의존성 설치 및 크롤러 실행
  - 갱신된 SQLite 데이터베이스 및 `index.html` 결과물을 프라이빗 깃 레포지토리에 자동 Git Commit & Push 처리
  - 정상 Push 완료 후 텔레그램 브리핑 최종 발송 트리거 연동
- [ ] GitHub Repository Secrets에 텔레그램 봇 토큰 및 개인 챗ID 안전 저장 연동

### 통합 테스트
- 로컬 CLI 환경과 GitHub Actions 가상 환경에서 수집, 분류, 파일 갱신, 커밋 및 텔레그램 발송까지의 전체 일련의 프로세스가 오류 없이 끝까지 이어지는지 확인하는 엔드투엔드(E2E) 가동성 확보
