# GameFinanceScraper -- 기획 및 디자인 문서 가이드

> 이 폴더는 게임회사 재무 채용공고 정밀 분석 및 프라이빗 텔레그램 브리핑 서비스의 핵심 비즈니스 요구사항과 설계 방향성을 기록한 기획 산출물 세트입니다.
> **Show Me The PRD (oh-my-claudecode)**를 통해 완벽히 조율되고 엄격한 템플릿 검증을 완료했습니다.

---

## 문서 구성 및 네비게이션

수동 탐색이나 AI 코더와의 완벽한 공동 협업을 위해 문서들이 역할에 맞게 정밀 분리되어 있습니다.

| 순서 | 문서 | 핵심 내용 | 읽어야 하는 타이밍 |
| :--- | :--- | :--- | :--- |
| 1 | **[01_PRD.md](./01_PRD.md)** | 서비스 개요, 타겟 사용자 정의, 핵심 가치, MVP 우선순위 기능 정의, Out of Scope 목록 | 기획의 핵심 목적과 뼈대를 이해할 때 |
| 2 | **[02_DATA_MODEL.md](./02_DATA_MODEL.md)** | 수집 마스터, 정밀 분류 테이블, 실행 로그 SQLite DB 엔티티 설계 및 1:1 관계 정형화 | 데이터베이스 구축 및 데이터 수집을 설계할 때 |
| 3 | **[03_PHASES.md](./03_PHASES.md)** | Phase 1 (수집), Phase 2 (하이브리드 분류 및 HTML), Phase 3 (텔레그램 및 자동 배포) 로드맵 및 가이드 프롬프트 | 단계별 점진적 개발을 개시할 때 |
| 4 | **[04_PROJECT_SPEC.md](./04_PROJECT_SPEC.md)** | AI 코딩 행동 강령, 기술 스택 결정 근거, 절대 하지 마(DO NOT) 목록, 테스트 및 무비용 환경변수 운영 방법 | AI 개발 에이전트에게 구현 작업을 직접 위임할 때 |

## 다음 개발 단계로 넘어가는 법

본 기획 문서를 기반으로 실제 개발을 바로 개시하려면 **[03_PHASES.md](./03_PHASES.md)**의 `"Phase 1 시작 프롬프트"` 블록 전체를 복사하여 에이전트(혹은 코드 생성 도구)에게 전달하시면 즉시 MVP 구축 개발 단계로 안전하게 교량 역할을 할 수 있습니다.

## 미결 사항 종합 (Needs Clarification)

기획 고도화 및 첫 개발 가동을 개시하면서 해결해야 할 최종 미결 사안 리스트입니다.

1. **공식 채용 페이지 스크래핑 차단 시나리오**: 게임사 공식 사이트의 방화벽(Cloudflare 등) 차단 감지 시 대응을 위해 수동 데이터 입력 폼 혹은 크롤링 오프라인 폴백 방식을 제공할지 여부 ([01_PRD.md](./01_PRD.md) 및 [04_PROJECT_SPEC.md](./04_PROJECT_SPEC.md) 참조)
2. **공고 변동 분석(MODIFIED)의 세부 텍스트 매칭 기준**: 연도나 단순 오타 수정을 제외하고, 실제 업무 내용 및 우대 조건에서 중요한 키워드 델타가 감지되었을 때만 '수정' 알림을 보낼 것인지에 대한 판정 수치 정의 ([02_DATA_MODEL.md](./02_DATA_MODEL.md) 참조)
3. **회사 메타 정보 갱신 연산 최적화**: 기업 규모(사원수, 매출액)를 수집 때마다 실시간 외부 API 조회를 할 것인지, 캐싱을 거쳐 월 1회 갱신을 적용할 것인지 결정 ([02_DATA_MODEL.md](./02_DATA_MODEL.md) 참조)
