# GameFinanceScraper -- 데이터 모델

> 이 문서는 시스템에서 다루는 게임사 재무 채용공고의 구체적인 구조를 정의합니다.
> 비개발자도 직관적으로 이해할 수 있는 개념적 아키텍처와 상세 필드 정보를 기록합니다.

---

## 전체 구조

매일 수집되는 공고(`job_postings`)는 수집 로그(`scrape_logs`)와 연계되어 수치화되고, 인공지능 하이브리드 엔진을 거쳐 고부가가치 분류 데이터인 `job_categories` 정보와 1:1 관계를 형성합니다.

```
[scrape_logs] --1:N--> [job_postings] --1:1--> [job_categories]
```

- **실행 주기 기준**: 1개의 수집 주기(`scrape_logs`) 동안 여러 개의 채용공고(`job_postings`)가 새로 발견되거나 상태가 업데이트될 수 있습니다.
- **공고 기준**: 1개의 채용공고(`job_postings`)는 오직 1개의 정밀 분류 정보(`job_categories`)를 소유합니다.

---

## 엔티티 상세

### 1. job_postings (채용 공고 핵심 마스터)
채용 플랫폼 및 게임사 공식 사이트에서 직접 수집한 정적인 메타데이터 정보를 보관합니다.

| 필드 | 설명 | 예시 | 필수 |
|------|------|------|------|
| **id** (PK) | 플랫폼명과 원본 고유번호의 조합 | `wanted_12345` 또는 `nexon_2026_01` | O |
| **source** | 공고를 수집한 소스 출처 | `wanted`, `saramin`, `nexon`, `krafton` | O |
| **company_name**| 채용을 진행하는 게임 회사명 | `넥슨코리아`, `크래프톤`, `데브시스터즈` | O |
| **title** | 공고의 최종 제목 | `재무회계 담당자 채용 (결산 및 세무)` | O |
| **origin_url** | 원본 공고로 바로 연결되는 하이퍼링크 | `https://wanted.co.kr/wd/12345` | O |
| **location** | 실제 근무지 위치 | `경기 성남시 분당구 (판교)` | X |
| **posted_at** | 공고가 등록된 날짜 (미제공 시 수집일) | `2026-05-18` | O |
| **status** | 현재 공고 활성화 상태 | `ACTIVE` (모집중), `CLOSED` (마감), `MODIFIED` | O |
| **raw_html** | 정밀 파싱 및 재가공용 원문 데이터 (텍스트) | `[상세 텍스트 내용 전체]` | O |
| **first_seen_at** | 시스템에 처음 수집되어 기록된 일시 | `2026-05-21 08:00:23` | O |
| **last_updated_at**| 최근 변동사항이 감지되거나 갱신된 일시 | `2026-05-21 08:00:23` | O |

### 2. job_categories (AI 하이브리드 정밀 분석 결과)
수집된 공고의 텍스트를 분석하여 연봉, 재택여부, 회사 규모 등 세부적이고 가독성이 뛰어난 핵심 정량 지표들을 정제해낸 결과입니다.

| 필드 | 설명 | 예시 | 필수 |
|------|------|------|------|
| **job_id** (FK, PK)| `job_postings` 테이블의 id 참조 | `wanted_12345` | O |
| **primary_category**| 회계/세무/재무 등의 최종 전문 대분류 | `회계` (또는 `세무`, `재무/자금`, `내부통제`) | O |
| **min_experience**| 채용에서 요구하는 최소 연차 (0은 신입) | `3` (3년차 이상 의미) | X |
| **max_experience**| 요구하는 최대 연차 제한 (없을 시 NULL) | `7` (7년차 이하 의미) | X |
| **salary_min** | 분류해낸 최소 연봉 범위 (단위: 만원) | `4500` | X |
| **salary_max** | 분류해낸 최대 연봉 범위 (단위: 만원) | `6000` | X |
| **work_type** | 재택근무 시행 형태 및 일수 | `하이브리드 (주2회 재택)` 또는 `풀재택`, `전면출근` | O |
| **company_revenue**| 수집 기준 회사의 연 매출 규모 (단위: 억) | `1200` (1200억 원) | X |
| **company_size** | 수집 기준 회사의 총 사원 수 (인력 규모) | `350` (350명) | X |
| **key_requirements**| 필수 자격 요건 핵심 리스트 (JSON 형식) | `["K-IFRS 회계 결산 가능자", "더존 ERP 사용 가능"]` | O |
| **preferred_skills**| 우대 요건 핵심 리스트 (JSON 형식) | `["게임업계 유경험자", "영어 회화 우수자"]` | O |
| **tools_used** | 사용하거나 요구하는 주요 회계/재무 도구들 | `SAP`, `DOUZONE`, `Excel` (콤마 구분 또는 JSON) | X |
| **ai_summary** | 사용자가 직관적으로 볼 수 있는 한글 3줄 요약| `1. 판교 근무 3-5년 결산 담당\n2. 주 2회 하이브리드 재택\n3. SAP 활용 우대` | O |

### 3. scrape_logs (수집 및 알림 히스토리 관리)
매일 실행되는 자동 크롤링 작업에 대한 성능 수치 및 가동 기록을 보관하여 시스템의 투명성을 확보합니다.

| 필드 | 설명 | 예시 | 필수 |
|------|------|------|------|
| **run_id** (PK) | 자동 증가 인덱스 | `42` | O |
| **run_date** | 스크랩이 성공적으로 완료된 날짜 | `2026-05-21` | O |
| **newly_added** | 당일에 새롭게 발굴되어 추가된 공고 개수 | `4` | O |
| **modified_count**| 우대사항 변경 등 기존 공고가 수정된 개수 | `1` | O |
| **closed_count** | 공고 모집 종료가 확인되어 마감된 개수 | `2` | O |
| **is_success** | 에러 없이 전체 단계가 잘 완료되었는지 여부 | `1` (성공), `0` (실패) | O |
| **error_log** | 실패했을 때 디버깅을 위한 원인 에러 로그 | `Saramin Scraper Connection Timeout Error...` | X |

## 왜 이 구조인가

### 1. 1:1 관계의 완전 분리 설계 (`job_postings` <-> `job_categories`)
- **이유**: 크롤링하는 원본 원시 데이터(`job_postings`)와 AI/분석 엔진을 거친 도메인 특화 메타 정보(`job_categories`)를 철저히 나눕니다. 
- **장점**: 
  1. 수집 파이프라인에서 오류가 생기더라도 정제 파이프라인은 독립적으로 동작해 이전 수집 데이터에 영향을 주지 않습니다.
  2. 추후 로컬 분류 분석 기준을 개선하거나 AI 요약 프롬프트를 바꿀 경우, 원본 데이터를 다시 긁어올 필요 없이 `job_postings.raw_html` 컬럼을 읽어와 `job_categories`만 갱신하는 로컬 오프라인 작업이 아주 손쉬워집니다.

### 2. 가로폭 축소화 및 JSON 활용
- **이유**: `key_requirements`나 `preferred_skills`와 같은 텍스트 목록 데이터를 고정 테이블 컬럼 대신 유연한 JSON 형태 또는 경량 텍스트 포맷으로 담습니다.
- **장점**: SQLite 데이터베이스를 아주 단순하고 작게 유지하며, 단일 파일 HTML을 생성할 때 복잡한 다중 조인(Join) 쿼리 없이 한 번에 데이터 맵핑을 구현하여 뷰어 로딩 속도가 획기적으로 향상됩니다.
