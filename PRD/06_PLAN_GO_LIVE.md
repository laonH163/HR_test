# GameFinanceScraper 고도화 및 안정성 보강 작업 계획

본 작업은 무비용 자동 크롤링 시스템의 수집 누락률을 최소화하고, 분류 및 알림 전송의 신뢰도를 실제 서비스 상용 서비스 수준으로 보강하기 위한 작업 계획입니다.

## 1. 주요 게임사 및 계열사 수집 채널 추가 (누락 0% 방어)
현재 수집 대상에서 누락되어 있던 유망 중견 게임사 및 핵심 계열사들을 추가하여 잡코리아 우회 수집을 활성화합니다.
*   **컴투스홀딩스** (`1547703`): 컴투스 그룹의 상장 지주사이자 개별 재무 조직을 가진 핵심 채널
*   **위메이드맥스** (`16154318`) / **위메이드플레이** (`16152331`): 위메이드 그룹 산하 상장 게임 개발 및 서비스 법인들
*   **스마일게이트RPG** (`16152335`): 로스트아크 독자 법인으로 대형 재무 채용이 발생하는 독립 조직
*   **하이브IM** (`16153723`): 하이브 산하 게임 퍼블리싱/개발 자회사
*   **빅게임스튜디오** (`16154089`): 유망 서브컬처 중견 개발사

## 2. 텔레그램 데일리 브리핑 메시지 4,096자 제한 대응 (Message Chunking)
*   수집된 공고가 몰리는 성수기에 전송 텍스트가 4,096자를 넘어가면 텔레그램 API에서 전송 실패 에러를 뱉고 당일 알림이 유실됩니다.
*   `TelegramSender`에 최대 3,500자 이하 단위로 메시지를 문단 경계에서 안전하게 분할하는 **Chunking** 로직을 도입하고 연속해서 전송하도록 개선합니다.
*   오늘의 신규 공고가 일정 수(예: 8건)를 넘어설 경우, 가독성을 위해 상세 정보는 대시보드 링크를 참고하도록 컴팩트 템플릿으로 가변 스위칭 처리합니다.

## 3. 하이브리드 분류기 계열사 역매칭 폴백(Fallback) 구현
*   `company_meta_presets`에 매출/사원수 정보가 없는 계열사가 수집되었을 때, 지주사나 모회사의 데이터를 유추하여 대체할 수 있는 지능형 역매칭 로직을 제공합니다.
*   예: `위메이드맥스`, `위메이드플레이` -> `위메이드` 프리셋 준용. `넥슨게임즈` -> `넥슨` 프리셋 준용. `스마일게이트RPG` -> `스마일게이트` 계열 규모 유추 등.

## 4. HTML 대시보드 다크모드 대응 및 오프라인 UX 안정화
*   정적 HTML 대시보드 파일인 `templates/dashboard_template.html`에 브라우저/OS 시스템 설정을 감지하는 **자동 다크모드 테마**를 추가합니다.
*   외부 인터넷 연결이 약한 오프라인/로컬 구동 환경에서도 Tailwind UI 렌더링이 깨지지 않도록, CSS 에셋 캐싱 메커니즘을 튜닝하고 폰트 로딩 시 레이아웃 무너짐을 방어합니다.

## 5. 완벽한 안전성 검증 및 유닛 테스트 보강
*   추가된 어댑터 목록과 분할 메시지 발생 로직, 분류기 역매칭의 정확도를 완벽하게 검증하는 테스트 시나리오를 `tests/` 디렉토리에 보완하고 전체 테스트를 수행합니다.
