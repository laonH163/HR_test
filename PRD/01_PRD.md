## 1. 제품 개요

### 한 줄 요약
게임 회사의 재무(회계, 세무, 재무/자금) 직군 채용공고를 다양한 채용 플랫폼과 기업 채용 사이트에서 자동 수집하고 연차, 연봉, 재택 여부, 회사 규모 등을 정밀 분류하여 텔레그램과 HTML 리포트로 개인에게 매일 브리핑해 주는 무비용 자동 크롤링 서비스입니다.

### 해결하는 문제
- **정보의 파편화**: 원티드, 사람인, 잡코리아 등의 플랫폼뿐만 아니라 넥슨, 크래프톤, 엔씨소프트 등 대형 게임사들은 자체 채용 홈페이지에만 재무 채용공고를 올리는 경우가 많아 매번 개별 검색해야 하는 번거로움이 있습니다.
- **분류의 비효율성**: 공고 제목만으로는 담당 도메인(세무, 내부회계, 자금, 원가회계 등)이나 필수 연차, 매출액 기준 회사 규모, 재택근무 시행 여부 등을 한눈에 파악하기 어렵습니다.
- **실시간 확인의 어려움**: 매일 새로운 공고나 마감 공고의 변동사항을 수동으로 파악하기 힘들어 지원 타이밍을 놓치는 경우가 발생합니다.

### 핵심 가치
- **운영비 0원 자동화**: 추가적인 서버 유지 비용이나 AI API 호출 비용 없이, GitHub Actions와 로컬 하이브리드 분류기만을 활용해 완전 무비용으로 매일 24시간 작동합니다.
- **게임업계 & 재무 직군 특화**: 타 산업군 공고를 완전히 필터링하고 오직 게임사 재무 직군 정보만을 초정밀 수집·가공합니다.
- **완벽한 사생활 보장(Privacy)**: 상시 오픈된 대시보드 웹서버 대신 개인 텔레그램 봇으로 신속하게 알림을 받고, 상세 데이터 표는 내 PC의 로컬 HTML 파일로 완벽하게 비공개 보관하여 열람합니다.

## 2. 사용자

### 주요 사용자
- **누구**: 게임 업계로 이직을 희망하거나 커리어를 쌓고자 하는 재무/회계/세무/자금 전문가 및 구직자
- **상황**: 매일 수시로 여러 채용 사이트를 뒤지며 공고를 체크하기에는 본업으로 인해 시간적 여유가 부족하고 피로감을 느끼는 상황
- **목표**: 나에게 맞는 게임사 재무 공고(예: 연차 적합, 원치 않는 출근 없는 하이브리드 근무, 일정 규모 이상의 매출액 기업)를 놓치지 않고 빠르게 포착하여 지원하는 것

### 사용자 시나리오
1. 사용자는 최초 한 번 자신의 텔레그램 봇 토큰과 사용자 ID를 설정해 둡니다.
2. 매일 오전 지정된 시간에 백그라운드(GitHub Actions)에서 수집 엔진이 동작하여 플랫폼과 주요 게임사 공식 홈을 스크래핑합니다.
3. 수집된 공고는 하이브리드 분류 규칙에 의해 연봉, 재택여부, 회사 규모, 연차 정보가 정밀 태깅됩니다.
4. 분석 완료 즉시, 사용자의 텔레그램으로 "신규 공고 2건, 수정 1건, 마감 3건" 등의 **데일리 브리핑 요약 카드**가 전송됩니다.
5. 상세 정보나 전체 누적 목록이 필요한 경우, 갱신된 로컬 HTML 파일을 실행하여 풍부한 검색 및 필터링 기능이 제공되는 대시보드로 분석 결과를 쾌적하게 확인합니다.

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름
```
Github Actions 스케줄 실행 -> 공고 스크래핑 -> AI 하이브리드 분류 -> Delta 분석 -> 텔레그램 메시지 전송 및 HTML 빌드 -> 로컬 HTML 확인
```

### 상세 흐름
1. **GitHub Actions 스케줄러**: 매일 오전 8시(한국 시간)에 워크플로우가 자동으로 가동됩니다.
2. **Scraper Engine**: 원티드, 사람인 API/웹페이지 및 넥슨, 엔씨소프트, 넷마블, 크래프톤 등 게임업계 대기업의 공식 채용 사이트를 동시 크롤링합니다.
3. **Classification Engine**: 수집된 텍스트 데이터를 분석하여 세부 카테고리(세무/회계/재무 등), 요구연차, 예상 연봉, 재택근무 조건(재택가능여부 및 일수), 회사정보(매출액/사원수)를 분류합니다.
4. **Delta Analyzer**: 로컬 SQLite 데이터베이스와 연계하여 새롭게 등장한 공고, 정보가 바뀐 공고, 내려간 공고를 분류합니다.
5. **Report & Alert**:
   - 요약 카드 형태의 포맷티드 텍스트가 텔레그램 봇 API를 통해 사용자의 모바일 폰으로 발송됩니다.
   - 전체 누적 및 당일 수집 데이터를 정돈한 `index.html` 단일 웹 대시보드 파일을 자동 빌드하여 리포지토리에 저장합니다.
6. **로컬 뷰어**: 사용자는 휴대폰 알림으로 동향을 확인한 후, 상세 분석이 필요할 때 PC에서 `index.html` 파일을 더블클릭하여 강력한 테이블 필터(예: '재택', '3년 이상', '매출액 1000억 이상')를 활용해 검토합니다.

## 5. 성공 기준

- [ ] **무보수 구동**: 사용자가 추가적인 인프라 비용 결제 없이 완전 무료로 매일 1회 이상 정상 자동 구동될 것
- [ ] **수집률 95% 이상**: 주요 4대 채용 플랫폼 및 5대 대형 게임사 공식 채용 페이지의 재무 관련 공고 누락이 없을 것
- [ ] **알림 정시 도착**: 매일 지정된 오전 시간대에 텔레그램 알림 메시지가 오차 범위 10분 이내로 수신될 것
- [ ] **무오류 빌드**: 대시보드 역할을 수행하는 HTML 파일은 외부 CDN 라이브러리에만 의존하며, 단독 오프라인/로컬 실행 시 자바스크립트 에러 없이 반응형 동작이 가능할 것

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 기능은 첫 단계(Phase 1, 2)에서 구현하지 않습니다.

- **범용 직군 스크래퍼**: 기획자, 개발자, 디자이너 등 타 직군 공고 수집 (이유: 수집 범위와 AI 분석 연산 과부하 방지 및 정밀도 극대화를 위해 오직 '재무/세무/회계' 직군만 한정)
- **자체 상시 웹서버 및 회원가입 로그인 체계**: (이유: 완전 무비용 서버리스 아키텍처를 유지하고 개인 프라이버시를 보장하기 위해 별도의 백엔드 호스팅 인프라 및 DB 호스팅을 구축하지 않음)
- **스크래핑 우회용 유료 프록시 인프라**: (이유: 운영비 0원 제약을 충족하기 위해 유료 프록시 서비스를 이용하지 않으며, 차단 회피는 헤더 튜닝 및 요청 주기 간격 조절 등 무료 기법으로 제한함)

## 7. [NEEDS CLARIFICATION]

> 기획 및 설계 단계에서 명확히 짚고 가야 할 미결 사항입니다.

- [ ] **공식 채용 페이지 스크래핑 주기 관리**: 일부 대형 게임사 공식 홈페이지는 보안 방화벽(Cloudflare 등)을 적용하고 있을 수 있으므로, 크롤러 차단 시나리오 발생 시의 예외 처리 및 수동 크롤링 입력 대체 지원 여부
- [ ] **스크랩 대상 주요 게임사 범위 정의**: 우선순위가 가장 높은 탑티어 5대 게임사(넥슨, 크래프톤, 엔씨소프트, 넷마블, 카카오게임즈) 외에 추가 수집할 중견 게임사 풀 리스트업 범위
- [ ] **로컬 AI 분류 라이브러리 가용 자원**: GitHub Actions 러너 성능 제약(RAM 7GB 제한 등) 내에서 무리 없이 동작할 수 있는 초경량 자연어 분석 알고리즘 또는 로컬 AI 모델 선택 구체화
